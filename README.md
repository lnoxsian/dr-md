# dr.md

dr.md is a native Linux Markdown editor and previewer built with Rust, `egui`, and `eframe`. It is designed to stay lightweight while still covering the day-to-day workflow of editing notes, browsing a folder-based vault, previewing Markdown, and keeping the UI responsive.

## What It Does

- Opens a folder as a vault and browses it in a file explorer sidebar.
- Edits Markdown with a rope-based text buffer and undo/redo support.
- Renders live preview with `egui_commonmark`.
- Supports editor, preview, and split views.
- Persists theme and workspace preferences in a local config file.
- Uses native file dialogs, file watching, and standard keyboard shortcuts.

## Quick Start

```bash
cargo run
```

On first launch, dr.md opens with a folder picker. After you open a vault, the app remembers the last folder in its config file.

## Build

For the full build guide, see [docs/BUILD.md](docs/BUILD.md).

```bash
cargo build
cargo run --release
cargo test
cargo clippy --workspace --all-targets --all-features -- -D warnings
```

If you prefer task wrappers, the repository also includes `make` and `just` targets that mirror the same commands.

## Project Layout

```text
src/            Application code
crates/         Local patched dependencies
assets/         App artwork and generated icons
assets/logo/    App logos and in-app branding
assets/icons/   Window icons and app icon sizes
docs/           User and developer documentation
scripts/        Asset generation helpers
```

## Assets

App logos live in [assets/logo](assets/logo) and icons live in [assets/icons](assets/icons). The shipped logo files are `dr-md-logo.png`, `dr-md-inapp-light.png`, and `dr-md-inapp-dark.png`. The icon set includes a `.ico` file plus multiple PNG sizes for native desktop packaging.

The logo and icon assets are generated by the helper script in `scripts/generate_assets.py`.

For the full asset reference, see [docs/ASSETS.md](docs/ASSETS.md).
